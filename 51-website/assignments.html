Problem set grading standards summary (subject to modification for future assignments):

Submissions are graded according to the following guidelines:

5-point submissions: all code is correct except for possibly one very minor mistake. Style is universally clean and careful except for possibly a couple of minor style issues or up to one larger style issue.

4-point submissions: no more than two total "mistakes", where a mistake is defined as incorrect code in some part of a function or two style issues.

3-point submissions: the midpoint of the scale, a 3-point submission is defined by being better than a 2-point submission, but not up to the standards of a 4-point submission (i.e., it exhibits more than two of the issues mentioned above). In general, such submissions will have entirely correct code in the majority of the exercises, but errors of both correctness and style scattered throughout.

2-point submissions: almost all functions have some correct cases, but almost all functions contain some mistakes.

1-point submissions: many exercises not attempted and/or the majority of exercises have incorrect behavior on most inputs.

Note in particular that 5-point submissions will be rare. A very good submission is likely to earn a 4; a fairly good submission is likely a 3.
