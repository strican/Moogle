Objectives

In this assignment, you will get a chance to continue to solidify your understanding of Ocaml's type system and get more practice working out complex function definitions. You will also continue working with algebraic datatypes, as you will need to understand how to work with a programmer-defined binary_tree type and write several higher-order functions that operate on values of that type. You will also for the first time need to design your own type, an n-ary tree, as a follow-up to the binary tree question. Finally, you will once more need to figure out and rewrite bad code.

To begin, download the assignment tarball, and extract it. For this assignment, you will be working on three different files:

  • 42.ml contains code for each of the exercises that you will do in Part 1.
  • trees.ml contains declarations for the functions that you will implement in Part 2.
  • style.ml contains the code that you will be rewriting in part 3.

As in the previous assignments, all of your programs must compile. Programs that do not compile will receive an automatic zero. Make sure that the functions you are asked to write have the correct names and the number and type of arguments specified in the assignment. Finally, please pay attention to style and follow the style guidelines posted on the course web site. Think carefully before writing the code, and try to come up with simple, elegant solutions.

Part 1: Reasoning about Ocaml programs (42.ml)

These functions are intended to help you learn how to reason about code. Think about the substitution model that Greg presented in lecture in attempting to decipher what the code we've given you is doing. Think carefully about types. Try to reason though these without using the compiler, and only use the compiler to test once you're pretty sure of your answer. We can tell you for sure right now that you'll be asked to do these on exams when you won't have a compiler handy, and practicing them on paper now is the best preparation you can give yourself.

Part 2: Datatypes and higher order functions (trees.ml)

The objective of this part is to learn how to think about programmer-designed datatypes. You will first build binary tree analogs to the fold and map functions we're already familiar with, and use them to implement a few handy functions that operate on binary trees. You will then design your own datatype to represent n-ary trees and implement a few higher-order functions over this new type.

Part 3: rewriting bad code (style.ml)

As you have been asked to do before, you will in this part need to take poorly written code, understand what it does, and rewrite it more efficiently. We strongly encourage you to take a global view of this task from the beginning! Don't start out by trying to make the code locally prettier; instead sit down and figure out what the code is doing at a high level before deciding to rewrite it. The style of your solution will count towards its correctness value, since this problem is all about style (the original code works as intended).
