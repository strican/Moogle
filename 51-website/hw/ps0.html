Instructions

The purpose of this assignment is to get you set up for the real assignments to come. You will set up an editing environment for Ocaml, make sure you can compile and run some toy programs, and figure out the assignment submission system.

Part 1: Signing up for a bulletin board account

Go to forums.seas.harvard.edu/cs51, click Register (top left), and sign up for an account. If you have problems with any of the rest of this assignment (or any of the other assignments), post to this bulletin board.

Part 2: Installing Ocaml

  • Ocaml is already installed on the FAS nice servers. You can use it there (run ocaml), but we'd recommend also installing it on your own computer. To do so, download the appropriate package for your operating system from the Ocaml download page and then follow the directions. Windows users: we recommend first installing the latest release of Cygwin, and then getting the "MinGW-based" port of Ocaml (listed first on the Ocaml download page).
  • To test that you can get Ocaml working on the FAS machines, use SSH to login to nice.fas.harvard.edu:
    • From a terminal, type ssh userid@nice.fas.harvard.edu, using the correct userid of course.
    • From a graphical ssh client, follow the directions :)
    Now run ocaml.
  • To test locally, just run ocaml from a terminal ("Command Prompt" in Windows). Whether you're running on the server or locally, you should now see the Ocaml prompt: 
    $ ocaml
        Objective Caml version 3.11.1

#

    If you type 

    "are we there yet?";;
    you should get the output: 
    - : string = "are we there yet?"

    Congratulations! If you got this far, Ocaml works. Type 

    #quit;;
    or Ctrl-D to exit Ocaml. Now you just have to learn how to write useful programs.
  • If you prefer not to use a terminal-based text editor to complete your work, we have also provided instructions for getting OCaml running in Eclipse, which is a free IDE that will also work for Java.

Part 3: Downloading the code and filling out the survey

Click here to download ps0.tgz. Save it somewhere, run tar -xzf ps0.tgz to unzip it (or use your favorite GUI program like winzip). As above, you can work on nice.fas or locally on your computer. Save and unzip the file where you intend to do your work.

You should now have a directory with a ps0.ml file inside. Your job is to edit this file, setting all the variables to appropriate values.

If you are using Eclipse, you will have to create a new Ocaml Managed Project (see instructions above for using Eclipse) for PS0. Copy the files you downloaded and unzipped into the folder Eclipse creates for your new project. From within Eclipse, select File -> Refresh. You should now see the ps0 files added to your Eclipse project, in the Navigator pane. You can then edit ps0.ml by double clicking it.

Once you're done, make sure the file still compiles: if you're using Eclipse, use the instructions for that. If you're using emacs, use the provided Makefile by typing make in the ps0 directory: 

$ make
ocamlc -o survey ps0.ml

$ ./survey

Check that the values are all filled in correctly. Once they are, you just have to submit.

Part 4: Submitting

To submit the assignment, your assignment directory must be on nice.fas. If you were working on your computer, upload the directory containing ps0.ml and the other downloaded files to nice. To upload, you'll need an FTP client (see instructions page, under "Uploading your work to nice").

Run make submit. You should get output like the following: 

$ % make submit
./submit.sh
Ok, submitting the files in /home/s/h/shnayder/release/ps0-submit
Creating temporary directory /tmp/shnayder.submit...
Copying assignment files to /tmp/shnayder.submit...
Submitting lib51 assignment 0... Done!
Temporary files removed.

Done!
$

If you've submitted successfully, you're done!
