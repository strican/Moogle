Here are some links to interesting articles or blog posts that are related to cs51. If you find some of the ideas in them interesting, feel free to come talk to Greg or any of the rest of staff about them. You are not expected to necessarily understand everything linked here :)

  • http://blogs.msdn.com/shawnhernan/archive/2010/02/13/microsoft-s-many-eyeballs-and-the-security-development-lifecycle.aspx.

    A discussion about whether open source makes for inherently more secure software.

  • http://cacm.acm.org/magazines/2010/2/69354-a-few-billion-lines-of-code-later/fulltext.

    A great article about static code analysis. My favorite quote:

    "Law: You can't check code you can't parse. Checking code deeply requires understanding the code's semantics. The most basic requirement is that you parse it. Parsing is considered a solved problem. Unfortunately, this view is naïve, rooted in the widely believed myth that programming languages exist."

    Slides from an interesting talk by Guy Steele (Java, Scheme)--how to organize functional code for parallel execution.

  • http://blogs.msdn.com/ericlippert/archive/2010/02/04/how-many-passes.aspx

    A fun post about the steps involved in compiling a real language (C#).
