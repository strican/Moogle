Installing OCaml, Java and dependencies

  1. Windows only: Install the latest version of Cygwin.
  2. Download and install the latest release of Objective Caml for your operating system. Windows users should get the "MinGW-based" port (listed first), some of whose features require Cygwin.
    NOTE: It looks like some people are having trouble with version 3.11.2 on OS X. Try 3.11.1 instead if 3.11.2 doesn't work.
  3. Make sure that you are running either version 1.5 or version 1.6 of the Java Virtual Machine. If you don't know, go to http://java.com/en/download/help/testvm.xml. If your setup is correct, your version will show up as either Java 5 or Java 6. If needed, the latest JVM can be downloaded from http://java.com/en/download/index.jsp

Using the Ocaml toplevel

Ocaml gives you a way to interact with code that you've written called a "Toplevel" (read more about toplevels at http://en.wikipedia.org/wiki/Read-eval-print_loop). If you're using Emacs with tuareg-mode, you can easily enter the toplevel (see Emacs instructions). Once we get to using modules, we'll show you a way to do this within Eclipse/OcaIDE. For now, you can simply open up the program "Objective Caml" that was installed in step 2, and either (1) File -> Open the .ml file containing your code, or (2) copy and paste your code from Eclipse and hit enter. Now you'll be able to interact with the code that you have pasted in.

Uploading your work to nice

Windows users: Download and install WinSCP, an FTP client for Windows. Run WinSCP and login to your nice account. You can now drag and drop your local files (left panel) to your nice account (right panel), and vice versa.

Mac/Linux users: You can just use the command line scp, or find one of the many GUI versions. From the command line: 

$ scp -r ps0/ userid@nice.fas.harvard.edu:~/
